## Что пользователь уже отдельно попросил по сметам
1. Количество позиции должно меняться прямо в строке.
2. Нельзя ставить количество больше, чем реально есть в активной базе.
3. Технику нужно группировать по категориям, чтобы быстро понимать, хватает ли базы на смену.
4. Цена должна редактироваться прямо в строке.
5. Кнопку отдельного изменения цены убрать.
6. Таблицу сметы визуально выровнять: сейчас строки местами сидят неровно.

## Налог и скидки — текущая целевая логика
Пользователь описал желаемую модель:

### Налог
- по умолчанию `0%`
- пресеты:
  - `ИП +9%`
  - `СЗ +6%`
- плюс свой налог вручную

Налог:
- задается на уровне проекта,
- отображается в проекте и в смете,
- считается после скидки,
- формула:
  - `цена - скидка + налог`

Если налог не нулевой, в смете должна быть отдельная строка:
- `Включая налог ...`

### Скидка
Пользователь хочет иметь:
- скидку проекта,
- скидку сметы,
- чтобы новые сметы могли наследовать скидку проекта.

## Что делать на новом компьютере
1. Установить:
- git
- node
- npm
- gh

2. Клонировать репозиторий:

```bash
git clone https://github.com/raeses/rental-crm.git
cd rental-crm
git pull origin main
```

3. Открыть в Codex именно:
- папку `rental-crm`

4. Если нужно продолжать PR workflow:

```bash
gh auth login
```

5. Для проверки окружения:

```bash
git --version
node -v
npm -v
gh --version
```
